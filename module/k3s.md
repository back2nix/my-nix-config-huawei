# K3s Local Configuration

Этот модуль настраивает K3s для использования `10.0.0.1` в качестве IP-адреса ноды. Это необходимо для стабильной работы в мобильных средах (например, при переключении между сетями Wi-Fi на ноутбуке), чтобы избежать сбоев сервисов, зависящих от hostNetwork (например, Ingress Nginx).

## Примененная конфигурация

В `module/k3s.nix` добавлены следующие параметры:
- `networking.interfaces.lo.ipv4.addresses` = `10.0.0.1/32`
- `--node-ip 10.0.0.1`
- `--node-external-ip 10.0.0.1`
- `--flannel-iface lo`

## Решаемая проблема
При смене IP-адреса интерфейса (через DHCP) K3s может продолжать использовать старый IP, что приводит к отказу liveness/readiness проб и падению подов. Использование `10.0.0.1` на `lo` гарантирует стабильность, так как Kubernetes запрещает использовать `127.0.0.1` в качестве Node IP.
